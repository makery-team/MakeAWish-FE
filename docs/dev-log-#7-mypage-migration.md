# Dev Log #7: 마이페이지 하단 탭 통합 및 UI 수정

## 작업 내용 (Migration & UI Fix)

1. **상단 구형 마이페이지 제거**
   - 파일: `components/header.tsx`
   - 내용: 우측 상단에 위치했던 프로필(User) 아이콘을 제거하고, 해당 아이콘을 통해 열리던 하단 모달형 마이페이지 기능을 삭제했습니다.
   - 사유: 하단 네비게이션 탭(`app/(tabs)/mypage.tsx`)이 신설됨에 따라 중복 접근 경로를 단일화하기 위함입니다.

2. **사용하지 않는 모달 컴포넌트 정리**
   - 삭제된 파일: `components/my-page.tsx`
   - 내용: 기존 모달 UI 방식의 마이페이지를 코드베이스에서 완전히 제거하여 레거시 코드를 청소했습니다.

3. **새 마이페이지 라우팅 연결**
   - 파일: `app/(tabs)/mypage.tsx`
   - 내용: 기존 구형 마이페이지에 있던 주요 기능들을 새 마이페이지의 메뉴 항목으로 이주했습니다.
     - "주문/예약 내역" 클릭 시 `/orders` 경로로 이동
     - "찜한 목록" 클릭 시 `/favorites` 경로로 이동
     - "내 리뷰 관리" 클릭 시 `/reviews` 경로로 이동

4. **알림창(Notification Panel) 위치 오프셋(UI) 수정**
   - 파일: `components/header.tsx`
   - 내용: User 아이콘이 사라지면서 우측 상단에는 알림(Bell) 아이콘 하나만 남게 되었습니다. 기존에 아이콘 2개를 기준으로 계산되었던 알림 패널 위치(`right: -40`)로 인해 알림창이 우측으로 치우치는 버그가 발생했습니다.
   - 수정: 알림창 컨테이너의 `right` 속성을 `0`으로 수정하여 화면 정중앙에 올바르게 위치하도록 조정했습니다.

## 향후 작업
- 마이페이지 내 주문 내역, 찜한 목록 카운트 등 실제 데이터 바인딩 고도화
- 추가적인 마이페이지 내 고객센터 및 공지사항 등 세부 화면 구현
