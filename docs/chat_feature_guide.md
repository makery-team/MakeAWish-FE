## 🏗️ 1. 어떤 파일들이 추가되었나요?

채팅 기능을 위해 크게 4가지 역할을 하는 파일들이 추가되었습니다.

1. **화면(UI):** 
   - `app/(tabs)/chat.tsx`: 하단 '채팅' 탭을 눌렀을 때 나오는 **채팅방 목록 화면**입니다.
   - `app/chat/[roomId].tsx`: 특정 채팅방을 클릭하고 들어갔을 때 나오는 **실시간 대화방 화면**입니다. (카카오톡 대화창 같은 곳이에요!)

2. **API 통신 (서버와 데이터 주고받기):**
   - `services/chat.ts`: 채팅방 목록을 가져오거나(`GET`), 방을 삭제(`DELETE`)하는 등 서버와 통신하는 함수들이 모여있는 곳입니다.

3. **실시간 통신 (웹소켓):**
   - `hooks/useChatSocket.ts`: 백엔드와 파이프를 연결해서 실시간으로 카톡을 주고받게 해주는 **웹소켓(WebSocket)** 전용 커스텀 훅입니다.

4. **타입 정의:**
   - `types/index.ts`: 채팅방(`DirectChatRoom`)과 메시지(`DirectChatMessage`) 데이터가 어떻게 생겼는지 모양(타입)을 정의해 둔 곳입니다.

## 🛠️ 2. 핵심 로직 살펴보기

### A. 일반 API 통신은 어떻게 하나요? (`services/chat.ts`)
우리 프로젝트는 토큰(로그인 증표)을 넣어서 통신하는 `fetchWithAuth`라는 유용한 함수를 씁니다.
채팅방 목록을 가져오는 코드를 보면 아주 간단합니다.
```typescript
async getChatRooms(): Promise<DirectChatRoom[]> {
  // 1. 서버에 GET 요청을 보냅니다.
  const res = await fetchWithAuth('/api/chatting/rooms');
  // 2. 에러가 나면 에러를 던집니다.
  if (!res.ok) throw new Error('Failed to get chat rooms');
  // 3. 서버가 준 데이터를 자바스크립트 배열(JSON)로 바꿔서 리턴해줍니다!
  return res.json();
}
```

### B. 실시간 웹소켓(WebSocket) 통신은 어떻게 작동하나요? (`useChatSocket.ts`)
일반 HTTP 통신은 클라이언트(앱)가 "데이터 줘!" 라고 할 때만 서버가 답을 줍니다. 하지만 채팅은 상대방이 글을 쓰면 **내 화면에 자동으로 스윽- 떠야 하죠?** 그래서 서버와 항상 연결된 '웹소켓(파이프)'을 씁니다.

1. **`connect()`**: 채팅방에 들어오면 서버의 `ws://.../chats` 주소로 파이프를 연결합니다.
2. **`ws.onmessage`**: 파이프를 통해 서버에서 메시지가 슝~ 날아오면, `setMessages`를 이용해 내 화면(배열)에 메시지를 쏙! 추가해 줍니다. 그럼 화면에 말풍선이 나타납니다.
3. **`sendMessage()`**: 내가 글을 쓰고 전송 버튼을 누르면, 이 파이프(`wsRef.current.send()`)를 통해 서버로 내 메시지를 던집니다.

### C. 에러 처리는 어떻게 되어 있나요?
백엔드 서버가 잠시 죽었거나 아직 개발이 덜 돼서 에러가 날 수 있습니다.
이때 앱이 튕기면(크래시) 안 되기 때문에 `app/(tabs)/chat.tsx`의 `loadRooms` 함수를 보면 `try...catch` 문으로 안전하게 감싸두었습니다. 에러가 나도 앱이 죽지 않고 빈 화면을 띄워줍니다!

## 💡 3. 코드를 수정할 때 주의할 점

1. **날짜 표기법 (Date)**
   원래 `date-fns`라는 라이브러리를 쓰려고 했으나, 앱이 무거워지고 꼬이는 것을 방지하기 위해 **순수 자바스크립트 Date 객체**를 사용하도록 변경했습니다. 시간을 수정할 때는 `toLocaleTimeString`이나 `toLocaleDateString` 함수를 사용해 주세요!

2. **Optimistic Update (낙관적 업데이트)**
   `[roomId].tsx`에서 내가 메시지를 전송할 때, 서버에서 "성공!" 응답이 올 때까지 기다리지 않고 **내 화면에 먼저 말풍선을 그려버립니다.** (이를 낙관적 업데이트라고 부릅니다). 덕분에 사용자는 앱이 훨씬 빠르다고 느끼게 됩니다!
