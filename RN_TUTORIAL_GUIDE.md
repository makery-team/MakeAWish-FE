# 🎓 [MakeAWish-FE] 입문자를 위한 React Native 가이드 & 프로젝트 해설

이 문서는 React(Web) 지식만 있는 상태에서 React Native(RN) 환경의 본 프로젝트를 이해하고 확장하기 위한 튜토리얼 가이드입니다.

---

## 1. Web 개발자가 반드시 알아야 할 RN 핵심 문법

### 1) HTML 태그 대신 전용 컴포넌트 사용

RN은 브라우저가 없기 때문에 `<div>`, `<span>`, `<img>`를 사용할 수 없습니다.

- **`<View>`**: `<div>`와 같습니다. 레이아웃을 잡는 기본 단위입니다.
- **`<Text>`**: 모든 텍스트는 반드시 이 태그 안에 있어야 합니다. (`<span>`, `<p>` 역할)
- **`<TouchableOpacity>`**: `<a>`나 `<button>` 대신 사용합니다. 누르면 투명도가 변하는 피드백을 줍니다.
- **`<Image>`**: `<img>` 대신 사용하며, `source={{ uri: 'url' }}` 형태로 데이터를 받습니다.

### 2) 스타일링 (Flexbox의 차이점)

- **`flexDirection`의 기본값은 `column`**: 웹은 가로(row)가 기본이지만, 모바일은 세로(column)가 기본입니다.
- **단위 없음**: `px`, `rem` 대신 숫자만 씁니다. (예: `padding: 16`) 이는 기기의 해상도에 맞춰 자동으로 계산되는 '단위 독립적 픽셀'입니다.
- **`StyleSheet.create`**: CSS 파일 대신 JS 객체로 스타일을 정의합니다. 성능 최적화에 도움이 됩니다.

### 3) 특수 레이아웃 대응

- **`SafeAreaView`**: 아이폰의 '노치'나 하단 바 영역에 UI가 겹치지 않게 보호해줍니다.
- **`KeyboardAvoidingView`**: 키보드가 올라올 때 입력창이 가려지지 않게 자동으로 밀어 올려줍니다.

## 2. 우리 프로젝트의 핵심 컴포넌트 로직 해설

### 📂 `AISearchBar.tsx` (바텀 시트 & 제스처)

이 프로젝트에서 가장 어려운 부분이지만, 이해하면 RN의 정수를 깨달을 수 있습니다.

- **핵심 기술**: `Reanimated` (애니메이션) + `Gesture Handler` (손가락 추적).
- **로직 설명**:
    1. 사용자가 시트를 잡고 위로 올리면 `translateY` 값이 실시간으로 변합니다.
    2. 손을 떼는 순간 속도(`velocity`)를 계산하여, 자석처럼 특정 위치(`half` 또는 `full`)에 착 붙게 만듭니다.
    3. **왜 이렇게 어렵게 하나요?** 웹처럼 단순 `height` 변경을 쓰면 느리고 끊기기 때문입니다. RN은 별도의 'UI 스레드'에서 애니메이션을 돌려 60fps의 부드러움을 유지합니다.

### 📂 `CakeGrid.tsx` (리스트 최적화)

- **핵심 기술**: `FlatList`.
- **로직 설명**:
  - 웹에서는 `.map()`을 써서 리스트를 그리지만, 모바일에서는 수천 개의 데이터를 한꺼번에 그리면 앱이 죽습니다.
  - `FlatList`는 **화면에 보이는 부분만 먼저 그리고** 나머지는 스크롤할 때 실시간으로 생성/제거하는 아주 똑똑한 컴포넌트입니다.

### 📂 `app/(tabs)/_layout.tsx` (파일 기반 라우팅)

- **핵심 기술**: `Expo Router`.
- **로직 설명**:
  - `app/` 폴더 안의 폴더 구조가 곧 앱의 메뉴가 됩니다.
  - `(tabs)` 폴더는 하단 탭 메뉴를, `shop/[id].tsx`는 상세 페이지를 담당합니다. 웹의 `Next.js`와 매우 흡사한 방식입니다.

## 3. 이 프로젝트를 공부하는 팁 (Step-by-Step)

1. **`constants/theme.ts` 확인**: 우리 앱의 '색상표'와 '폰트'가 어떻게 정의되어 있는지 보세요.
2. **`components/header.tsx` 수정해보기**: 로고 텍스트를 바꿔보거나 아이콘 위치를 옮겨보며 `StyleSheet`에 익숙해지세요.
3. **`hooks/use-navigation.ts` 읽어보기**: "상태(State)" 하나로 어떻게 지도가 보였다가 리스트가 보였다가 하는지 그 흐름을 추적해보세요.
4. **콘솔 로그 활용**: RN도 `console.log()`를 쓸 수 있습니다. 터미널 창(Expo 서버)에서 로그를 확인하며 데이터가 어떻게 흐르는지 보세요.
